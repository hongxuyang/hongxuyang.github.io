NAT/PAT
IPV4地址紧缺 将私网地址转换为公网地址
Debug ip nat ——抓取测试中的发送地址、转换条目、目的地址
将路由器实现PC机的功能
pc2(config)#no ip routing
pc2(config)#int e1/0
pc2(config-if)#ip add 192.168.1.2 255.255.255.0
pc2(config-if)#no shut
pc2(config)#ip default-gateway 192.168.1.3
测试拓扑：
左边两个路由器分别PC1PC2，连接到交换机，交换机再连接到路由器【中介路由】，再连接到外部路由器。内网网段192.168.1.0/24.外网网段202.100.34.0/24.
R1R2都是PC，分别PC1、PC2 R4配loopback口 4.4.4.4/24

静态：一对一
第一、 确定ipnat的内部、外部
第二、 将私网IP与公网IP做关联
第一、
R3(config)#int e1/0
R3(config-if)#ip nat side
R3(config)#int e1/1
R3(config-if)#ip nat outside
第二、
R3(config)#ip nat inside source static 192.168.1.1 202.100.34.100
现在R1可通R4
R3(config)#ip nat inside source static 192.168.1.2 202.100.34.101
现在R2也可通R4 202.100.34.4
【ip nat inside source static + 源+目的】
如何查看对应的地址？
R3#show ip nat translations

动态：一对一转换
第一、 确定ipnat的内部、外部
第二、 确定感兴趣流的ACL（其实就是需要转换的私网IP地址）【192.168.1.1—192.168.1.127，ACL内输入192.168.1.1 0.0.0.127】
第三、 确定公网地址池（Pool池）
第四、 将感兴趣流的ACL与Pool此做关联
第一步和静态第一步一样
R3(config)#access-list 1 permit 192.168.1.0 0.0.0.127【第二步】
R3(config)#ip nat pool ?【第三步】
WORD Pool name

R3(config)#ip nat pool WOLF ?
A.B.C.D Start IP address
netmask Specify the network mask
prefix-length Specify the prefix length

R3(config)# ip nat pool WOLF 202.100.34.100 202.100.34.110 netmask 255.255.255.0【ip nat pool +地址池名字+从哪个网段+到哪个网段+子网掩码】
<1-2699> Access list number for local addresses
WORD Access list name for local addresses

R3(config)#ip nat inside source list 1 pool WOLF【最后一步】

多对一： PAT
第一、 确定ipnat的内部、外部
第二、 确定感兴趣流的ACL
第三、 将感兴趣流的ACL与outside接口做关联+overload
Ip nat inside source list 1 int e1/1【边界路由器与外面连接的接口】 overload
多对少：
第一、 确定ipnat的内部、外部
第二、 确定感兴趣流的ACL
第三、 确定公网地址池pool池 【R3(config)# ip nat pool WOLF 202.100.34.100 202.100.34.110 netmask 255.255.255.0【ip nat pool +地址池名字+从哪个网段+到哪个网段+子网掩码】】
第四、 将感兴趣的ACL与pool池做关联+overload【R3(config)#ip nat inside source list 1 pool WOLF overload】
与动态1对于1唯一区别就是做关联+overload

NAT应用：
第一种、
ip nat inside static tcp 192.168.1.1 23 202.100.34.200 23 extendable
ip nat inside static tcp 192.168.1.1 23 202.100.34.200 10000 extendable
左边映射成右边
可站在公网去访问内网的服务器 其实就是包含了端口的应用

第二种、
公网访问两台服务器。基于TCP的两台服务器的负载均衡
确定IPNAT内部、外部
确定感兴趣流的ACL access-lists 2 permit 202.100.34.200
确定地址池pool池 （两台server的地址）192.168.1.1 192.192.1.2 type 选择rotary【轮训】【不会做池上去看红字】
感兴趣流的ACL与Pool池做关联 –>ip nat inside destination list 2 pool WOLF2

用户登录服务器会发现1与2可以轮流登录。

WAN
DTE——————DCE
时钟频率确保两端一直 在DCE设置
Show controllers +接口 可看DCE还是DTE
点到点 的 交换机 进接口Clock rate 6400在DCE端设置 DTE敲上不会生效

点到点默认封装为HDLC
PPP封装会推送对端接口路由 两边不在同一网段，一样能通
PPP如何封装？
R1(config)#int s1/0
R1(config-if)#encapsulation ppp
Mar 31 22:54:29.323: %LINEPROTO-5-UPDOWN: Line protocol on Interface Serial1/0, changed state to down Mar 31 22:54:37.483: %LINEPROTO-5-UPDOWN: Line protocol on Interface Serial1/0, changed state to up
两边都做完封装 接口会up起来
R1(config-if)#ppp authentication ?
chap Challenge Handshake Authentication Protocol (CHAP)
eap Extensible Authentication Protocol (EAP)
ms-chap Microsoft Challenge Handshake Authentication Protocol (MS-CHAP)
ms-chap-v2 Microsoft CHAP Version 2 (MS-CHAP-V2)
pap Password Authentication Protocol (PAP)

PAP认证：
R1(config-if)#ppp authentication pap【使用pap实现ppp认证】
*Mar 31 22:56:36.935: %LINEPROTO-5-UPDOWN: Line protocol on Interface Serial1/0, changed state to down
R1(config-if)#ppp pap sent-username R1 password 123 【自己的用户名和密码】
做双向认证，实测可通

HasH->md5 定长128bits、不可逆、雪崩效应【无论多相似的数据，运算后没有一点点相似】

CHAP认证：
也是基于PPP封装的，进端口配！！！

R1(config-if)#encapsulation ppp
R1(config-if)#ppp authentication chap
R2(config-if)# encapsulation ppp
R2(config-if)#ppp chap host R2
R2(config-if)#ppp chap password 123
以上是完成的单向认证
