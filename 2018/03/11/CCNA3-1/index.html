第二层交换和生成树协议

第二层交换的三种不同功能：
第一：地址学习。第二层的交换机和网桥能够记住在某个接口上收到的每个帧的源硬件地址，而且它们还会将这个信息输入到被称为转发/过滤表的MAC数据库中
第二：转发/过滤决策。当某个接口上收到一个数据帧时，交换机就会在MAC数据库中查看其目的方的硬件地址，并找到其输出接口。此数据帧只会被转发到相应的目的端口
第三：环路避免。如果为了保证冗余而在交换机之间创建了多重连接，网络就可能产生回路。在提供冗余的同时，生成树协议可房子网络环路的产生

端口安全：如何房子不相干人员将电脑连接到交换机上？
默认情况下MAC地址是动态地出现在MAC转发/过滤数据库中，因此可使用端口安全阻止上述连接行为
例：Switch(config)#int f0/1
Switch(config-if)#switchport mode access
Switch(config-if)#switchport port-security ?
mac-address Secure mac address
maximum Max secure addresses
violation Security violation mode


由于所有思科最新交换机所配置的端口处于可取模式（当检测到其他交换机连接时，端口倾向于中继连接）。因此我们首先必须将端口的模式从可取模式修改为接入端口，否则就无法进行安全性的配置。

Switch(config-if)#switchport port-security mac-address
可以将独立的MAC地址分配到交换机的每个端口上，前提是保证你必须有充足的时间

Switch(config-if)#switchport port-security maximum 1
Switch(config-if)#switchport port-security violation shutdown
交换机每个端口只允许进入一台主机，当有人破坏这个规则时，端口就会关闭。（集线器进入）。Maximum 1表示端口只能使用一个MAC地址，如果超过就关闭，若要再开启只能手动输入 no shutdown 命令来恢复启用

Switch(config-if)#switchport port-security mac-address sticky
Switch(config-if)#switchport port-security maximum 2
Switch(config-if)#switchport port-security violation shutdown
主要是为静态MAC地址提供安全保障，借助它就无需在网络配置时输入每个MAC地址。 进入stick端口前两个地址被认定为静态地址，并且不管aging命令中设定多长时间，它一直会是静态地址。为什么设置2？一个PC，一个电话。

生成树协议STP——主要是防止第二层网络（网桥或者交换机）出现网络环路。它会监事网络以找出可用链路，并关闭任何冗余链路以确保不会出现环路。STP首先用生成树算法（STA）创建一个拓扑数据库，然后找出并关闭冗余链路。运行STP后，数据帧就可以在最优链路上转发！

其实就是找出所以链路，关闭冗余链路，阻止网络环路出现。

操作：
Switch(config)#spanning-tree vlan 1 priority ?

<0-61440> bridge priority in increments of 4096
Switch(config)#spanning-tree vlan 1 priority 4096
设置vlan 1中交换机的优先级为4096后，指定了这台交换机为根桥

生成树的端口快速，如果交换机上连接服务器或者其他设备，并且你完全确定这些链路不会因为禁用STP而产生交换环路。那么可在这些端口上使用端口快速，表明当STP汇聚时，这端口无需花费50秒即可进入转发模式
Switch(config-if)#spanning-tree portfast ?
disable Disable portfast for this interface
trunk Enable portfast on the interface even in trunk mode

批量进入端口快速模式
Switch(config-if)#int range f0/1 -12
Switch(config-if-range)#spanning-tree portfast

安全详细配置：
s1(config)#int range f0/1
s1(config-if-range)#switchport mode access
s1(config-if-range)#switchport port-security maximum ?

<1-132> Maximum addresses
s1(config-if-range)#switchport port-security maximum 1
s1(config-if-range)#switchport port-security mac-address sticky
s1(config-if-range)#switchport port-security violation ?
protect Security violation protect mode
restrict Security violation restrict mode
shutdown Security violation shutdown mode
s1(config-if-range)#switchport port-security violation shutdown
s1(config-if-range)#ex
要先range 进入端口，设置模式为access，默认为desirable，即如果他们发现自己连接到另外的交换机，它们倾向于中继。当端口处于desirable时无法设置安全端口。之后设置允许关联的MAC地址最多只有一个。而且只有第一个关联的MAC地址可以通过交换机发送帧。如果不同MAC地址的机器也向交换机发送帧，则端口由于设置violation命令，其实就是违规关闭。

然后可以使用 命令：show port-security interface 【端口号】 来验证端口的安全

s1(config)#int range f0/1
s1(config-if-range)#spanning-tree portfast ?
disable Disable portfast for this interface
trunk Enable portfast on the interface even in trunk mode


s1(config-if-range)#spanning-tree portfast
使用portfast命令，可以防止汇聚时间过长导致主机DHCP请求超时，从而使得主机不能应答DHCP地址的问题。使用时一定得小心。不然可能创建网络环路！！！

Bpduguard ——使用前如果交换机端口打开了portfast，那么，启用bpduguard是个不错的主意，如果启动bpduguard的交换机端口接收到一个BPDU，他会将端口设置为错误禁用状态。这就可以防止管理员不小心将另一台交换机或者集线器连接到配置了portfast的交换机端口上。基本上这一命令可以防止网络崩溃，至少防止网络因环路而性能下降。注意：只能在接入层交换机使用这一命令，因此接入层交换机是上的用户是和交换机直连的，而核心交换机则不能使用此命令

Bpdufilter——另一个可以与portfast同时使用的命令。由于默认情况下启用了portfast的交换机端口仍可以接受BPDU，这时使用bpdufilter可以组织BPDU从这一端口进出。如果bpdufilter接受到一个BPDU，他会立即关闭portfast，迫使端口重新成为STP拓扑的一部分，与bpduguard不用，它不会讲端口设置为错误禁用状态，而是将端口打开，但是不允许portfast。

所以建议启用portfast时候，端口建议同时启用bpduguard或者bpdufilter！！！
典型做法是只使用其中一个，因为bpdufilter接受BPDU后会将端口转换到非portfast状态。而bpdufilter接受BPDU后将端口设置为出错禁用状态。
例：s1(config)#int range f0/1
s1(config-if-range)#spanning-tree bpduguard ?
disable Disable BPDU guard for this interface
enable Enable BPDU guard for this interface
s1(config-if-range)#spanning-tree bpduguard enable

uplinkfast 是一个全局命令，在所有端口上启用
s1#conf t
s1(config)#spanning-terr uplinkfast
s1(config)#do show spanning-tree uplinkfast——————检验命令

backbonefast
s1(config)#spanning-tree backbonefast
s1(config)#do show spanning-tree backbonefast——————检查命令
uplinkfast——可以确定并快速修复本地交换机的链路失效
backbonefast———不只限于接入层交换机，可以用来确定远程交换机上非直接连接到根路径的链路失效

根桥看法：其实就是MAC地址最小的就是根桥（从左往右看）

设置根桥命令
core(config)#spanning-tree vlan 1 priority ?

<0-61440> bridge priority in increments of 4096
core(config)#spanning-tree vlan 1 priority 16384
值可以任意设置，如果设置为0 代表永远是根桥，设置为61440代表永远不可能是根桥
