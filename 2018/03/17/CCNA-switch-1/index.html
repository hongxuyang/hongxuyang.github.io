交换机：
学习 MAC
泛红广播、组播、未知单播（相同VLAN的接口和相同trunk的接口）【trunk默认情况下可以允许所以vlan】
已知单播，点到点转发

一个接口可以关联多个MAC地址

遇到MAC地址泛洪攻击————可以在端口设安全，比如只学习3个MAC地址，超过就自动关闭，或者丢弃流量。

不同VLAN的接口，交换机知道MAC地址表，也不会传输数据。

Vlan
1 默认vlan
2-1001 正常配置 1002-1005系统保留 1006-4094扩展VLAN

如何创建：
sw1(config)#vlan 10
sw1(config-vlan)#name CCNAAAA
sw1(config-vlan)#vlan 20
sw1(config-vlan)#name AAAAA
或者：
sw1#vlan database
% Warning: It is recommended to configure VLAN from config mode,
as VLAN database mode is being deprecated. Please consult user
documentation for configuring VTP/VLAN in config mode.

sw1(vlan)#vlan 301 name 11111
VLAN 301 added:
Name: 11111
sw1(vlan)#apply
APPLY completed

如何查看：
Show vlan brief
如果要删除 前面加上no

如何将vlan划分给接口
sw1(config)#int f0/1
sw1(config-if)#switchport mode access 【access 是接入接口】
sw1(config-if)#switchport access vlan 10
查看命令
sw1#show run int f0/1

如何批量端口操作：
sw1(config)#int range f0/30-35

当删除vlan时候，需要把接口初始化，
sw1(config)#default interface range f0/30-35【default 初始化】

trunk
可以承载所有VLAN，它是一种二层封装，将两端成为trunk而已。
封装方式：ISL【思科私有】、802.1q
ISL【inter switch link】：不会改变数据帧。从交换机出来，进入trunk。数据帧头部增加26字节ISL的头部，以及4个字节的CRC校验字段。用来表示属于哪个vlan，通过trunk数据到了交换机，可以通过头部的数据，来知道数据属于哪个VLAN。

802.1q：会改变数据帧。目的和源和MAC的地方插入tag字段，4个字节。在tag内表示属于哪个VLAN，在帧到达目的交换机的时候拿掉目的和源MAC
优点：只增加4个字节

Native vlan 只能有一个 不能做任何标记
两端要保持 native vlan一致 默认是vlan 1

Trunk如何设置：
sw1(config)#int range f0/11-12
sw1(config-if-range)#switchport trunk encapsulation ? 【封装】
dot1q Interface uses only 802.1q trunking encapsulation when trunking
isl Interface uses only ISL trunking encapsulation when trunking
negotiate Device will negotiate trunking encapsulation with peer on
interface
sw1(config-if-range)#switchport trunk encapsulation dot1q 【封装成动态dot1q】
sw1(config-if-range)#
22:12:57: %LINEPROTO-5-UPDOWN: Line protocol on Interface FastEthernet0/11, changed state to down
22:12:58: %LINEPROTO-5-UPDOWN: Line protocol on Interface FastEthernet0/12, changed state to down
sw1(config-if-range)#switchport mode trunk【再将端口起来】【相当于是强制开启】

如何查看：
sw1#show interfaces trunk
Port Mode Encapsulation Status Native vlan
Fa0/11 on 802.1q trunking 1
Fa0/12 on 802.1q trunking 1

如何更改native vlan 【两端都要做，而且要保持一致】
sw1(config)#int range f0/11-12
sw1(config-if-range)#switchport trunk native vlan 20

只让设置的vlan通过trunk【一端做了就行】
sw1(config)#int range f0/11-12
sw1(config-if-range)#switchport trunk allowed vlan 20

DTP ————动态trunk协商协议。使得交换机与交换机之间自动协商trunk。
两种模式：
Desirable：主动发DTP，主动和对端协商同一个trunk，收到对方发送的消息也可以做回应
Auto：被动的，收到以后可以回应
Mode trunk：强制开启trunk
Mode access：强制关闭trunk
两端不同模式情况下，trunk不能起来的情况：
Desirable ——access ×
Auto——auto ×
Auto——access ×

例：
sw1(config)#do show inter trunk
Port Mode Encapsulation Status Native vlan
Fa0/11 desirable n-isl trunking 1
Fa0/12 desirable n-isl trunking 1
模式 n代表trunk起来了

sw1(config-if-range)#switchport nonegotiate 不再自动协商，禁止端口向对方发送协商包，可以说是个优化命令，通常与trunk模式联用

VTP——VLAN Trunking Protocol 同步交换机的VLAN信息。
前提：1、交换机之间必须起trunk
2、VTP的域名必须相同，默认情况下是空，不会同步其他的交换机，但是可以被同步
3、如果有VTP认证必须得互相通过

如何查看VTP信息：
sw1#show vtp status

Configuration Revision : 0 【配置修订号，每对vlan做一次操作，修订号会自动加1，无论新建或者删除】
Maximum VLANs supported locally : 1005
Number of existing VLANs : 21
VTP Operating Mode : Transparent
VTP Domain Name :

VTP V2 Mode : Disabled

如何设置VTP域名：【一定要创建域名，才会同步，域名不能为空】【改域名会使得修订号为0】
sw1(config)#vtp domain WOLF
Changing VTP domain name from NULL to WOLF
sw1(config)#do show vtp sta
VTP Version : 2
Configuration Revision : 0
Maximum VLANs supported locally : 1005
Number of existing VLANs : 22
VTP Operating Mode : Transparent
VTP Domain Name : WOLF

如何修改VTP模式：
sw1(config)#vtp mode ?
client Set the device to client mode.
server Set the device to server mode.
transparent Set the device to transparent mode.

VTP模式【实际情况中只将一两台交换机模式设置为server模式，防止误操作】
Server：默认的，可以创建修改vlan信息
Clien：不能增加修改vlan信息
Transparent：透明模式，支持扩展vlan。和其他交换机互不影响，互相不同步。修订号固定为0。
VTP同步的时候。交换机配置修订号高的同步配置修订号低的，如果相同直接忽略

Vlan信息保存在配置文件中，配置文件保存在NVRAM中
可以转发VTP消息

STP
两台机器之间，为了冗余（防止发生故障）。多设置一台交换机，但是可能发生重复的数据包、MAC-table不稳定（翻动）、广播风暴。
因此出现了生成树协议（STP）。可以确保冗余的同时也是无环的而且走的是最短的路径。

操作步骤
1、 网络选根桥
2、 非根桥设备选根端口
3、 选择DP指定端口
4、 关掉不要的端口
