第二层交换型网络最大好处是：与每个交换机端口相连的每台设备都是一个独立的冲突域，消除了以太网的密度约束，从而能够组建更大的网络。然而每项更新改进会带来新的问题，例如，网络包含的用户和设备越多，每台交换机需要处理的广播和分组也就越多。

另一个问题：安全。因为默认情况下每个用户都能看的所有设备，所以，唯一可采取的措施是在服务器和其他设备上设密码。

控制广播：每种协议都使用广播，但是广播频率取决于3个因素：
协议的类型。互联网中运行的应用程序。这些服务的使用方法。

静态VLAN：是创建VLAN的最常用方法，原因之一是静态VLAN最安全。这种安全源自：将交换机端口分配到特定VLAN后，除非手动修改，否则它一直属于该VLAN。且易于配置和管理，非常适合用于需要控制所有用户移动的网络环境。

动态VLAN：自动确定节点所属VLASN，可根据硬件地址（MAC），协议甚至创建动态VLAN的应用程序来确定节点所属的VLAN。动态接入口只能属于一个VLAN，是有VMPS分配的。在同一交换机中，可以同时有动态接入端口和中继端口，但动态接入端口只能连接到终端或集线器，而不能连接到其他交换机。

VLAN中继协议（VTP）
目标是：管理交换型网络中配置的所有VLAN，确保整个网络的一致性。可以添加、删除、重名VLAN。
必须提供的功能：
网络中所有交换机的 VLAN配置都必须一致 ;
让VLAN能够跨越不同类型的网络 ,如以太网和 ATM LANE(或 FDDI);
准确地跟踪和监视VLAN
随时将新增的VLAN报告给 VTP域中其他所有的交换机
以即插即用的方式新增 VLAN。

两台交换机要交换VLAN信息，必须满足下面三项：
两台交换机的 VTP管理域名相同
其中一台交换机被配置为VTP服务器
VTP密码相同 (如果使用了的话 )

服务器模式（Server）：
提供VTP消息：包括VLAN ID和名字信息。学习相同域名的VTP消息。转发相同域名的VTP消息。可以添加、删除和更改VLAN VLAN信息写入NVRAM。该模式下不能使用扩展

客户机模式（Client）
请求VTP消息。学习相同域名的VTP消息。转发相同域名的VTP消息。不可以添加、删除和更改VLAN VLAN信息不会写入NVRAM。该模式下不能使用增强型软件映像提供的VID，即只能使用2-1001这一段的值。1，1002-1005均为系统默认VID。要使用1006-4096作VID的值，只能关闭VTP或采用透明模式。

透明模式（Transparent）
不提供VTP消息。不学习VTP消息。转发VTP消息。可以添加、删除和更改VLAN，只在本地有效 VLAN信息写入NVRAM。

VTP修减：配置VTP以减少、组播、单播分组，从而节省带宽，这种被称为修剪。启用VTP修剪的交换机仅将广播发送给确实需要它的中继链路
Show interface trunk 可以查看是否禁用了VTP修剪

如何将交换机端口分配给VLAN？
s1(config)#int f0/3
s1(config-if)#switchport ?
access Set access mode characteristics of the interface
mode Set trunking mode of the interface
native Set trunking native characteristics when interface is in
trunking mode
nonegotiate Device will not engage in negotiation protocol on this
interface
port-security Security related command
priority Set appliance 802.1p priority
trunk Set trunking characteristics of the interface
voice Voice appliance attributes

上面新出了许多命令，access、mode、nonegotiate、trunk、voice

s1(config-if)#switchport mode ?
access Set trunking mode to ACCESS unconditionally
dynamic Set trunking mode to dynamically negotiate access or trunk mode
trunk Set trunking mode to TRUNK unconditionally
s1(config-if)#switchport mode access ————告诉交换机这是一个永久非中继的第二层接口
s1(config-if)#switchport access vlan 3————将端口分配给一个VLAN
要同时分配几个端口可以使用：interface range f0/1-0/3
现在就指定了端口所属的VLAN，但如果此时将设备连接到VLAN端口。它只能和同一个VLAN内的设备通信。下面将启用VLAN间通信。

如何配置成中继接口？
s1(config)#int f0/8
s1(config-if)#switchport mode trunk

一些常用命令解释：
s1(config-if)#switchport mode access————将接口永久设为非中继模式，并通过协商将链路设置为非中继链路。无论链路的另一端是否是中继接口，改接口都将为非中继接口，即专用的第二层接入端口。
s1(config-if)#switchport mode dynamic auto————让接口能够将链路转换为中继链路。如果邻接口为trunk或者desirable，改接口将成为中继接口。当前默认为dynamic auto。
s1(config-if)#switchport mode dynamic desirable————让接口尽力将链路转换为中继链路。如果邻接口为trunk或者desirable或者auto。该接口为中继接口。是以前一些交换机的默认模式。现在新的交换机中，所有以太网接口都默认采用这种模式。
s1(config-if)#switchport mode trunk————将接口永久设为中继模式。并通过协商将邻接链路转换为中继链路。即使邻接口不是中继接口，该接口也将成为中继接口
s1(config-if)#switchport nonegotiate————禁止接口生成DTP帧。仅当接口处于access或者trunk模式时候，才可以使用。这种情况下，要建立中继链路，必须手工将邻接口配置为中继接口

3560上面配置的中继：
core(config)#int f0/1
core(config-if)#switchport trunk encapsulation ?
dot1q Interface uses only 802.1q trunking encapsulation when trunking
core(config-if)#switchport trunk encapsulation dot1q
core(config-if)#switchport mode trunk

指定中继端口支持的VLAN：
core(config)#int f0/1
core(config-if)#switchport trunk allowed vlan ?
WORD VLAN IDs of the allowed VLANs when this port is in trunking mode
add add VLANs to the current list
all all VLANs
except all VLANs except the following
none no VLANs
remove remove VLANs from the current list
core(config-if)#switchport trunk allowed vlan remove ?

<1-1005> VLAN ID of a allowed VLAN when this port is in trunking mode
core(config-if)#switchport trunk allowed vlan remove 4
在0/1端口上配置中金链路，导致他丢弃来自VLAN4的数据。可以尝试将VLAN1排除在外，但中继链路仍将接受和发送管理数据流，如CDP、PAgP、LACP、DTP和VTP

如果要将特定范围内的VLAN排除在外：
core(config-if)#switchport trunk allowed vlan remove 4-8
将VLAN排除在外后，要恢复默认设置，可以使用下面命令：
core(config-if)#switchport trunk allowed vlan all
core(config-if)#no switchport trunk allowed vlan

修改中继端口本机VLAN：
core(config)#int f0/1
core(config-if)#switchport trunk ?
allowed Set allowed VLAN characteristics when interface is in trunking
mode
encapsulation Set trunking encapsulation when interface is in trunking mode
native Set trunking native characteristics when interface is in
trunking mode
core(config-if)#switchport trunk native ?
vlan Set native VLAN when interface is in trunking mode
core(config-if)#switchport trunk native vlan ?

<1-1005> VLAN ID of the native VLAN when this port is in trunking mode
core(config-if)#switchport trunk native vlan 40
core(config-if)#^Z
将本地端口VLAN修改为VLAN40后可用show running-config查看该口的配置
如下：
!
interface FastEthernet0/1
description 1st connect to s1
switchport trunk native vlan 40
switchport trunk encapsulation dot1q
switchport mode trunk
!
别忘了邻接的端口需要相互配合！我们可以将当前端口本机VLAN恢复到默认设置：
core(config-if)#no switchport trunk native vlan
这样当前中继端口将把VLAN作为本机VLAN。记住，中继链路两端本机VLAN必须相同！！！

VTP模式【实际情况中只将一两台交换机模式设置为server模式，防止误操作】
Server：默认的，可以创建修改vlan信息
Clien：不能增加修改vlan信息
Transparent：透明模式，支持扩展vlan。和其他交换机互不影响，互相不同步。修订号固定为0。
