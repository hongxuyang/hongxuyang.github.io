增强内部网关路由协议EIGRP是一个思科专用协议，只能用于思科路由器上
开放最短路径优先OSPF 路由选择协议，是目前最流行的路由选择协议

EIGRP 是一个无类、增强的距离矢量协议，使用了自治系统的概念来描述相邻路由器的集合，处于自治系统中的路由器使用相同的路由讯遏制协议并共享相同的路由选择信息。EIGRP非常适用于特大型网络应用，最大跳数为255，默认为100，EIGRP不会像RIP那样用跳数作为度量。对于EIGRP来说，跳数是用来限定路由更新数据包在被抛弃之前可以经过的路由个数。同样这个数字是限定AS的大小的，与如何计算度量无关

协议主要功能：
通过协议相关模块支持IP和IPv6(以及一些应用不广泛的其他被路由协议)
被认为是无类的（与RIPv2、OSPF一样）
支持VLSM/CIDR
支持汇总和不连续网络
高效的邻居发现
基于可靠传输协议 (RTP)的通信
基于弥散更新算法 (DUAL)的最佳路径选择

在EIGRP路由器彼此进行路由交换前必须成为邻居，要满足三个条件：
接受Hello 包、实现AS号的匹配、相同的度量（K值）

EIGRP的配置
要在路由器上启动一个 EIGRP会话 ,可以使用 router e1grp命令,并在其后指定网络的自治系统号（可以理解为一个区内）。然后,使用带有网络号的 network命令,输入连接到此路由器的网络号。
例：路由器连接两个网络10.1.3.0\24 172.16.10.0\24
Router(config)#router eigrp 20
Router(config-router)#network 172.16.0.0
Router(config-router)#network 10.0.0.0
注意：与使用RIP相同，这里使用的是有类的网络地址，即掩码中所以的子网和主机为均为0。
若要使得指定接口停止EIGRP的运行，需要使用passive-interface 【接口】
和RIP中的配置一样
例：Router(config)#router eigrp 20
Router(config-router)#passive-interface s0/0
意味着会将会阻止邻居关系，这个接口将不能再发送或者接受路由信息了

No auto-summary 命令可以关闭自动汇总功能，RIP上同样实用

Show ip route eigrp ——路由表中只显示EIGRP的项目

Show ip eigrp neighbors——只显示EIGRP的邻居 可以检查IP地址和重传间隔，以及已建立起邻接关系的邻居队列数

Show ip eigrp topology——显示EIGRP拓扑表中的表项 每个路由前面有个P，表明此路由处于被动状态，因为某个路由处于激活状态，则表明当前的路由器已经失去了通往这个网络的路径，并在搜索可替代的路径，每个表项给出了他指向远程网络的可行距离，即FD，以及数据包传递时需要经过的下一跳邻居。每个表项圆括号有两个数字，第一个用于指示可行距离，第二个是指通向远程网络的被通告距离

Show ip protocols——显示路由选择协议的配置

Debug eigrp packet——显示相邻路由器间发送接收hello数据包
Debug ip eigrp events——显示网络上EIGRP变化以及更新的事件

OSPF是一个开放标准的路由选择协议，被各种网络开发商使用，其中包括思科。首先，OSPF要构建一个最短路径树，然后使用最短路径的计算结果来组建路由表。OSPF汇聚很快但是没有EIGRP快，它同时支持对相同目标等价多路径路由，和EIGRP一样，支持IP和IPv6路由协议。

协议主要功能：
由区域和自治系统组成的框架
最小化路由选择的更新流量
允许可缩放性能
支持 VLSM/CIDR
拥有无限的跳计数
允许多开发商的设备集成 (开放的标准 )

OSPF是以分层结构来设计的,这代表你可以将大型的互联网络分割成一些被称为“区域”的小的互联网络。这也正是OSPF 在设计上的精华所在。
创建分层的原因：
减少路由选择的开销
加速汇聚
将网络的不稳定性限制在单一网络区域内

配置OSPF
1． Router ospf 1
2． Router-id【可选：设备自动会选择出router-id，便于网络管理和router-id冲突建议手动输入；使用设备编号来指定router-id】
3． Network+本地IP的网络号+网络号的通配符+area与编号
如果一个网络内两个直连IP地址 必须通告在一个area编号内
验证：1、邻居是否建立完成 #show ip ospf neighbor
2、查看路由表
3、查看接口ospf属性 # show ip ospf interface loopback 1
例：Router(config)#router ospf 1
Router(config-router)#router-id 1.1.1.1
Router(config-router)#network 10.0.0.0 0.255.255.255 area ?

<0-4294967295> OSPF area ID as a decimal value
A.B.C.D OSPF area ID in IP address format
Router(config-router)#network 10.0.0.0 0.255.255.255 area 0

还可以这样的方法：
直接输入自己连接其他路由器接口的地址 然后通配符输入0.0.0.0 后面跟上区域地址
IOU3(config-router)#router ospf 102
IOU3(config-router)#net 23.1.1.3 0.0.0.0 a 0
IOU3(config-router)#net 34.1.1.3 0.0.0.0 a 0

Debug ip ospf adj——显示广播和非广播多路访问上的DR和DBR选举过程

DR【指定路由器】 BDR【备份指定路由器】
BDR是有选举产生的用于防止DR失效的备份路由器，是要为路由器建立一个用于信息交换的中心连接点，这样更新数据的交换方式便从每个路由器都需要与网络分段中其他各个路由器进行交换，改版为路由器只需要发送它的信息给DR和BDR。随后再由DR将信息中转给每个路由器。

其中网络分段中带有最高OSPF优先级的路由器会成为本网络分段中的DR。优先级可以用how ip ospf interface 查看。默认值为1，如果所有路由器均使用默认优先级设置，这最高路由器ID(RID)的路由器会胜出。如果修改为0，就不会参加DR和BDR的选举。

RID是有OSPF启动时所有接口地址中最高的IP地址确定。使用环回接口可修改RID。

OSPF故障诊断：
两个相邻路由器，却无法建立邻接关系
命令：sh ip ospf interface 【相邻接口】
可看到双方的Hello 与 Dead时间不同，所以无法建立邻接关系

补充：
RIP&OSPF之间重分布
当一个路由协议重分布进入RIP时，RIP协议会认为新路由不是通过RIP数据包更新的信息【即OSPF更新包】，RIP会将这些路由信息开销设定为16，RIP开销（metric）:1-15【RIP更新次数】。超过15的RIP都认为不可达路由。因此任何协议从重分布进入RIP必须手动设定一个metric
Router rip
Redistribute ospf 1 metric 1
当路由协议重分布进入OSPF时，如果网络中的IP存在无类IP。那么此时命令必须携带subnets参数，如果不携带，OSPF只能发布有类路由
Router ospf 1
Redistribute rip subnets 可以显示开销值E2——不会变
Redistribute rip subnets metric-type 1 可以显示开销值E1——越远越大
